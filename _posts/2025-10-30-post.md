## TIL
### **DNS 동작 과정**
사용자 요청 및 로컬 캐시 확인 (사용자 -> DNS Resolver) 사용자가 웹 브라우저 주소창에 naver.com과 같은 도메인 이름을 입력한다.

사용자 PC는 먼저 자체 캐시에 naver.com에 대한 IP 주소가 있는지 확인한다. 
캐시에 정보가 없으면, 사용자 PC는 **로컬 DNS 서버(ISP의 DNS Resolver)**에 naver.com의 IP 주소 요청을 보낸다.

로컬 DNS 서버 (DNS Resolver): 보통 사용자의 **ISP(인터넷 서비스 제공자)**에서 제공하는 서버(ISP(공유기))이며, 사용자를 대신하여 IP 주소를 찾기 위해 여러 DNS 서버와 통신한다.

시작로컬 DNS 서버는 캐시에 찾는 정보가 없으면, IP 주소를 찾기 시작한다.

Root Server 질의로컬 DNS Resolver는 가장 상위 레벨인 Root Server에게 naver.com의 IP 주소를 요청한다. 

그 후 Root Server는 .com 도메인을 관리하는 TLD Server의 주소를 응답으로 알려준다다.

TLD Server 질의로컬 DNS Resolver는 응답받은 **TLD Server(.com)**에게 다시 naver.com의 IP 주소를 요청한다. 

TLD Server는 naver.com 도메인에 대한 최종 권한을 가진 Authoritative Name Server의 주소를 응답으로 알려준다. 

Authoritative Name Server 질의로컬 DNS Resolver는 응답받은 naver.com Authoritative Name Server에게 최종적으로 naver.com의 IP 주소를 요청한다. 

Authoritative Name Server는 자신의 영역인 naver.com의 실제 IP 주소를 로컬 DNS 서버에게 응답한다. 

IP 주소 반환 및 웹 서버 접속로컬 DNS Resolver는 응답받은 naver.com의 IP 주소를 캐시에 저장하고, 이 정보를 사용자에게 전달한다.

사용자는 로컬 DNS 서버로부터 전달받은 IP 주소를 이용하여 해당 서버(Server)에 TCP 3-way Handshake를 통해 연결을 시도한다. 

이때 TCP(80/443) 프로토콜을 사용하여 **웹 서버(Server)**에 HTTP/HTTPS 요청을 보낸다. 
 
Server는 요청을 처리하고 응답을 사용자에게 전송하며, 웹 브라우저는 해당 웹페이지를 화면에 표시한다.
